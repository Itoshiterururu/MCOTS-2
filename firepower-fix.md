# Виправлення розрахунку Firepower

## Проблема:
БПЛА, міномети та підтримуючі підрозділи мали firepower = 0-1 через:
1. Занадто жорстку формулу розрахунку
2. Занадто низький firepower_base для деяких типів

## Рішення:

### 1. Покращена формула:
```java
// Старо (давало 0-1):
firepower = firepower_base * (vehicles/10) * (supply/100) * (personnel/50)

// Ново (реалістичні значення):
firepower = firepower_base * max(0.3, vehicles/5) * max(0.5, supply/100) * max(0.4, personnel/30) * 5.0
```

### 2. Скориговані характеристики:
- **БПЛА**: firepower_base 1→2 (може атакувати дронами-камікадзе)
- **Інженерні**: firepower_base 3→4 (мають вибухівку)
- **Ремонт**: firepower_base 1→2 (мінімальна самооборона)
- **Постачання**: firepower_base 2→3 (охорона конвою)
- **Медичні**: firepower_base 1→2 (мінімальна самооборона)
- **Логістичні**: firepower_base 2→3 (охорона бази)

## Очікувані результати:

| Тип | Старий firepower | Новий firepower |
|-----|------------------|-----------------|
| Танки | 15-25 | 40-80 |
| Піхота | 10-20 | 35-70 |
| БПЛА | 0-1 | 8-15 |
| Міномети | 1-3 | 25-50 |
| Постачання | 1-2 | 12-25 |

## Логіка:
- **Бойові підрозділи** (танки, піхота) - найвища вогнева міць
- **Вогнева підтримка** (артилерія, міномети) - висока вогнева міць
- **Спеціальні** (розвідка, БПЛА) - помірна вогнева міць
- **Підтримка** (медичні, ремонт) - мінімальна але не нульова вогнева міць

Тепер всі підрозділи матимуть реалістичну бойову ефективність!
